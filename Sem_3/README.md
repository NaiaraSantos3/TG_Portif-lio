## Objetivo do Projeto

A instituição possui um sistema de processamento de dados em cascata chamado pipeline, que consiste em vários estágios que são coordenados automaticamente com base nas características das fontes de dados e nas necessidades dos clientes em termos de algoritmos de IA ou modelos matemáticos. 
Na metodologia de implementação do sistema, é necessário configurar as fontes de dados envolvidas para que a plataforma funcione corretamente. No entanto, a configuração das fontes de dados é feita manualmente, sendo um ponto crítico que consome muito tempo dos técnicos e cria uma dependência excessiva de especialistas para realizar tal tarefa.
Para superar esse obstáculo, foi proposto a criação de uma interface amigável para a configuração das fontes de dados em alguns estágios, proporcionando maior agilidade na implantação para os clientes da _Dom Rock_ e reduzindo a sua dependência de técnicos especializados.

## Contribuições Pessoais

Durante o desenvolvimento do DataFlow, atuei como Desenvolvedora Front-end, contribuindo diretamente para o design, implementação de interfaces e integração com o backend. Minhas principais entregas foram:

- Design e Prototipação
    - Participação ativa na idealização e criação dos protótipos das telas do sistema, garantindo alinhamento consistente com o cliente e com o time.
    - Suporte contínuo no ajuste e refinamento das telas ao longo do semestre, conforme feedback do cliente e das validações internas.


- Desenvolvimento Front-end
    - Implementação das principais telas do sistema, entre elas:
        - Tela Home, apresentando arquivos importados e informações filtradas por nível de acesso.
        - Tela de Login, incluindo integração com o backend para autenticação.
        - Tela de Validação Silver Zone, permitindo aprovação e reprovação de dados processados.
        - Tela de Edição DePara, possibilitando inclusão, exclusão e edição de metadados.
    - Realização da integração completa entre front-end e back-end, garantindo que as funcionalidades estivessem conectadas e operacionais.
    - Planejamento e registro das horas estimadas e realizadas para cada tarefa, acompanhando o fluxo de desenvolvimento do time.


- Funcionalidades Administrativas
    - Desenvolvimento e refinamento do Controle de Usuários, funcionalidade exclusiva do usuário Master, incluindo:
            - Cadastro de novos usuários;
            - Exibição do Log de Usuários;
            - Acesso às opções administrativas dentro do sistema.


- Documentação e Materiais de Apoio
    - Participação na criação de dois Manuais do Usuário, com conteúdos orientados a diferentes perfis:
        - Manual do Usuário – Parceiros: orientado aos clientes que utilizariam o sistema.
        - Manual do Usuário – Master e Full: destinado ao time interno responsável pela gestão da aplicação.


- Colaboração e GitHub
    - Utilização ativa do GitHub para versionamento, organização e acompanhamento das entregas do projeto.
    - Participação nas cerimônias ágeis, garantindo alinhamento contínuo com o time.
